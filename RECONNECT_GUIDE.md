# Автоматическое переподключение игроков

## Проблема
При обновлении страницы (например, при разблокировке телефона) игрок терял соединение и не мог продолжить участие в квизе.

## Решение
Добавлено автоматическое переподключение с использованием localStorage и распознавание игрока по имени на сервере.

## Как это работает

### 1. Сохранение сессии (клиент)
При успешном подключении к квизу сохраняется:
- Код сессии
- ID игрока
- Имя игрока
- Временная метка

### 2. Автоматическое переподключение (клиент)
При загрузке страницы `/play`:
- Проверяется наличие сохранённой сессии
- Если сессия найдена и не старше 30 минут
- Автоматически выполняется переподключение
- Показывается индикатор "Переподключение..."

### 3. Распознавание игрока (сервер)
При получении запроса на подключение:
- Проверяется, существует ли игрок с таким именем в сессии
- Если да - это переподключение:
  - Обновляется WebSocket соединение
  - Сохраняется прогресс (счёт, ответы)
  - Отправляется текущее состояние квиза
- Если нет - новый игрок (только в лобби)

### 4. Восстановление состояния
При переподключении игрок получает:
- **В лобби**: обычное подключение
- **Во время вопроса**: текущий вопрос с правильными вариантами
- **В результатах**: дождётся следующего вопроса
- **После завершения**: финальную таблицу лидеров

### 5. Очистка сессии
Сессия очищается:
- После отправки опроса (кнопка "Выйти")
- Если сессия старше 30 минут
- При ошибке переподключения

## Пользовательский опыт

### Сценарий 1: Обновление страницы во время квиза
1. Игрок участвует в квизе
2. Случайно обновляет страницу (F5 или pull-to-refresh)
3. Видит "Переподключение..."
4. Автоматически возвращается в квиз
5. Продолжает участие

### Сценарий 2: Блокировка телефона
1. Игрок ждёт следующего вопроса
2. Блокирует телефон
3. Разблокирует через несколько минут
4. Браузер обновляет страницу
5. Автоматическое переподключение
6. Игрок видит текущий вопрос или результаты

### Сценарий 3: Завершение квиза
1. Квиз завершён
2. Игрок отвечает на опрос
3. Видит "Спасибо за обратную связь!"
4. Нажимает "Выйти"
5. Сессия очищается
6. При следующем входе нужно ввести код заново

## Технические детали

### localStorage
```javascript
{
  "code": "ABC123",
  "playerId": "uuid-here",
  "name": "Иван",
  "timestamp": 1234567890
}
```

### Время жизни сессии
- 30 минут с момента подключения
- После этого требуется новое подключение

### Обработка ошибок
- Если сессия не найдена на сервере
- Показывается форма входа с сообщением об ошибке
- Игрок может войти заново

## Ограничения

### Что НЕ сохраняется
- Ответы на вопросы (они на сервере)
- Текущий счёт (восстанавливается с сервера)
- Состояние UI (восстанавливается при получении данных)

### Когда переподключение не работает
- Сессия завершена на сервере
- Прошло более 30 минут
- Сервер перезапущен (сессии в памяти)
- Очищен localStorage браузера

## Для разработчиков

### Отладка
```javascript
// Проверить сохранённую сессию
console.log(localStorage.getItem('playerSession'));

// Очистить сессию вручную
localStorage.removeItem('playerSession');

// Проверить возраст сессии
const session = JSON.parse(localStorage.getItem('playerSession'));
const age = Date.now() - session.timestamp;
console.log('Session age (minutes):', age / 60000);
```

### Тестирование
1. Войти в квиз
2. Открыть DevTools → Application → Local Storage
3. Проверить наличие `playerSession`
4. Обновить страницу (F5)
5. Должно произойти автоматическое переподключение
