# Отладка проблемы с вариантами опроса

## Проблема
Варианты множественного выбора отображаются в одну строку вместо отдельных кнопок.

## Как проверить

### 1. Откройте консоль браузера (F12)

### 2. Создайте вопрос с множественным выбором

В поле "Варианты" введите:
```
Вайбкодинг
Сетевые цензоры в России и мире
Как работает интернет
```

### 3. Нажмите "Сохранить"

В консоли должно появиться:
```json
Saving survey: {
  "questions": [
    {
      "type": "multi_choice",
      "text": "Какие темы интересны?",
      "options": [
        "Вайбкодинг",
        "Сетевые цензоры в России и мире",
        "Как работает интернет"
      ]
    }
  ]
}
```

### 4. Проверьте что сохранилось

Перезагрузите страницу и откройте редактор опроса.
Варианты должны отображаться каждый на своей строке.

## Если варианты всё равно в одну строку

### Проверка 1: Убедитесь что используете Enter
- НЕ запятые
- НЕ точки с запятой
- Только клавиша Enter между вариантами

### Проверка 2: Откройте test-survey-parsing.html
```bash
# Откройте в браузере
open test-survey-parsing.html
```

Вставьте ваши варианты и проверьте результат парсинга.

### Проверка 3: Проверьте БД
```bash
# Если используете SQLite напрямую
sqlite3 data/quizzes.db
SELECT survey FROM quizzes WHERE id = 'your_quiz_id';
```

Должно быть:
```json
{"questions":[{"type":"multi_choice","options":["Вариант 1","Вариант 2"]}]}
```

НЕ должно быть:
```json
{"questions":[{"type":"multi_choice","options":["Вариант 1\nВариант 2"]}]}
```

## Решение

Если проблема сохраняется:

1. Удалите квиз и создайте новый
2. Убедитесь что используете последнюю версию кода
3. Очистите кэш браузера (Ctrl+Shift+R)
4. Проверьте что сервер перезапущен после изменений

## Контрольный список

- [ ] Варианты вводятся каждый на новой строке
- [ ] В консоли видно правильный JSON при сохранении
- [ ] После перезагрузки варианты отображаются правильно
- [ ] При проведении квиза участники видят отдельные кнопки
- [ ] Каждая кнопка содержит один вариант
